Ch. 1: Values, Types, and Operators

Identify two similarities and two differences between JavaScript and Ruby syntax with regard to numbers, arithmetic, strings, booleans, and various operators.

Strings can be denoted with single or double quotation marks (”string” or ’string’) in both languages. In both languages, strings can also be concatenated literally (with + signs). E.g. "hel" + "lo"

In JavaScript, a triple equals sign (===) determines if two objects are the exact same object (same type and value). However in Ruby, a double equals sign (==) determines if two objects are the same object. A double equals sign in JavaScript determines if the values match so "1" == 1 would return true.

Incrementing and Decrementing is slightly different in JavaScript too. Where in Ruby it's x += 1, it's x++ in JavaScript.

In both languages basic arithmetic is the same. + denotes addition, - denotes subtraction, * denotes multiplication, / denotes division, and % will give the remainder of the operation. e.g. 1 + 2 = 3 or (3 + 4) * 5 = 35.

Boolean values in both languages are the same as well. They're true and false written out like that.

Logical operators are the same as well. && refers to logical and. || refers to the logical or. Logical operators can be combined with comparison operators (>, <, >=, <=, ==, !=) in both languages. The ternary operator can be used similarly in JavaScript to the way it is used in Ruby. e.g. console.log(true ? "true" : "false"); (the only thing that is different is the use of console.log to print to the console.)

Describe while and for loops

In a while loop, while some condition is true, the program will execute a statement. It will repeatedly execute that statement as long as the conditional expression is true. An example of a conditional expression is number < 5. Generally, to create a working while loop that isn't an infinite loop, some kind of counter variable will need to be created and then incrementally increase or decrease within the while loop.

A for loop in JavaScript includes the counter variable, the increment that the program will count by (e.g. every 2 --> 2, 4, 6, etc.), and the conditional statement all in the parentheses that initialize the for loop. Then after that first line, there's the actual statement(s) that you want to execute in the next line. The for loop takes up less lines than the while loop since it creates the counter variable, has the conditional statement, and the counting increment all in one line.

Ch. 3: Functions

What are the differences between local and global variables in JavaScript?

Variables declared outside of any function are global variables and are visible throughout the entire program. Local variables are created within a function and are local to that function.

When should you use functions?

A function is a block of code that is programmed to perform some kind of task. You should use functions when you want to re-use a block of code repeatedly to manipulate an object (or collection of objects).

What is a function declaration?

It is a statement that defines a variable and then points that variable at a given function. For example:

function double(x) {

  return x * 2

}

What is the difference between using a dot and a bracket to look up a property? Ex.array.max vs array["max"]

Both the dot and the square brackets access a property of array. A dot is must be used after a valid variable name and it names the property directly. However, for the square brackets, the expression within the square brackets is evaluated to get the property name. So array["max"] will evaluate "max" and use that result as the property name.

What are the biggest similarities and differences between JavaScript and Ruby?

JavaScript is much more finicky when it comes to how it interprets where statements begin and end. It is important to remember the semicolon for most statements so that JavaScript will evaluate one statement at a time instead of trying to lump it into a big statement. Where Ruby is able to interpret without extra help, it seems that JavaScript needs more help. For example, to declare a variable in JavaScript the programmer needs to preceed the variable name with var. In Ruby one just needs to type the variable name and it's original value. Both languages have built in methods like .push, .max, .min. Both languages have similar loops and allow variables (which can be integers, strings, arrays, or even functions) to be manipulated similarly. Basic arithmetic is the same.
